# Product Designer — Job Description & Hiring Scorecard

## Role Summary
Tourbillon is seeking a **Product Designer** to own the UI/UX of the platform, build a cohesive design system, and deliver polished components that elevate the user experience for our first 1,000 users.

## Responsibilities
- Design end-to-end user experiences for Tourbillon's core features (dashboard, onboarding, billing, integrations)
- Build and maintain a **Design System** (tokens, components, patterns) aligned with Next.js
- Lead the **Billing Dashboard UI** and **Onboarding Flow** design
- Collaborate with engineers to translate designs into React components
- Conduct user research and iterate based on Mixpanel analytics data
- Define design tokens (colors, typography, spacing, shadows) for theming
- Create wireframes, prototypes, and high-fidelity mockups in Figma
- Prepare the Product Hunt launch UI polish (cover image, badge, animations)

## Must-Have Skills
- **Figma**: Mastery of frames, components, variants, auto-layout, prototyping, and libraries
- **Design Tokens**: Ability to define and hand off tokens (color, typography, spacing) in JSON/CSS variables
- **React / Next.js**: Comfort working with components, Tailwind CSS or Radix UI
- **Responsive Design**: Desktop-first with mobile/tablet polish
- **User Research**: Synthesize qualitative/quantitative data into design decisions
- **Interaction Design**: Micro-interactions, loading states, tooltips, onboarding tours
- **Typography & Color Theory**: Strong visual hierarchy and accessibility awareness

## Nice-to-Have
- Experience designing SaaS dashboards or productivity tools
- Experience with design systems (e.g., Storybook, Radix UI, Headless UI)
- Motion/animations (Framer Motion, CSS transitions)
- A/B testing and increment-based design iteration
- Familiarity with Mixpanel or GA4 data

## Interview Process (3 Rounds)
1. **Portfolio Review & Live Design (60 min)**: Reviewedesign a real Tourbillon screen and explain decisions
2. **Design Challenge (90 min)**: Design and prototype the Billing Dashboard / Plan Selection flow in Figma
3. **Culture & Collaboration (60 min)**: Pair with an Engineer on a component design-to-code exercise

## Scorecard (Each Criterion: 1–5)
| Criterion | Weight | Description |
| --- | --- | --- |
| Visual Design Quality | ×3 | Clean, consistent, accessible interfaces |
| Figma & Design Tokens | ×2 | Efficient, well-structured Figma files; token discipline |
| UX Thinking | ×2 | Onboarding flows, interactions, whitespace, hierarchy |
| Frontend Fluency | ×1 | Component structure, responsive, design-to-code translation |
| Research-Driven | ×1 | Can synthesize data into design decisions |
| Collaboration | ×1 | Great comms with engineers; open to feedback |
| **Total Score** | **/22** | **Pass at ≥15** |
