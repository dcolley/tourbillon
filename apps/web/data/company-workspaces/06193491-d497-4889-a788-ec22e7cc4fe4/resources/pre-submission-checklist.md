# Pre-Submission Checklist for Authors

**Purpose:** All authors must confirm these items before submitting any Derek Colley content for review or posting. This prevents recurring voice/format issues that block publication.

---

## ✅ MANDATORY CHECKLIST

### Voice & Perspective
- [ ] **First person throughout** — All tweets written as if Derek himself wrote them (NOT third-person references to "Derek")
  - ❌ WRONG: "Derek already posts about scaling" / "This is why Derek's tweet resonated"
  - ✅ CORRECT: "I've been posting about scaling for months" / "I posted about this on..."

### Hashtags
- [ ] **At most 1 hashtag per tweet** — Preferably zero. Derek rarely uses hashtags.
  - If using one, choose the most relevant (#LocalAI is acceptable)
  - Never use: #OpenSource #AgenticAI #DGXSpark together (too many tags dilute impact)

### Technical Precision
- [ ] **Specific numbers included** — tok/s counts, model names (exact), hardware specs (1x DGX Spark, not "a GPU")
- [ ] **Real-world details** — Include at least one practical detail: OOM crash story, late-night bench run, config example, or similar authentic touchpoint

### Content Quality
- [ ] **No corporate language** — Avoid phrases like "we encountered resource constraints" → use "it crashed one DGX during compilation"
- [ ] **Self-deprecating/honest line included** — Derek shares failures openly (crashes, issues) not just wins
- [ ] **All primary source links present** — GitHub repos, HuggingFace models, papers, relevant X posts

### Format
- [ ] **Short paragraphs** — 1-3 sentences max per tweet block
- [ ] **Bullets/numbered lists for structured info** (→ arrows, 1️⃣2️⃣3️⃣)
- [ ] **Code/config snippets when relevant** (commands, model paths, hardware specs)

---

## 📝 SUBMISSION CONFIRMATION

When submitting content, include this line at the top of your comment:

> **Pre-submission checklist confirmation:**
> ✅ All items verified above before submission.

This creates accountability and makes review faster for Curator/Reviewer agents.
